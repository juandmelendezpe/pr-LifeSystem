Manual de Instalación

Guía de capacitación técnica para la configuración y levantamiento del entorno de desarrollo de LiFe System.

1. Requisitos Previos del Sistema

Antes de comenzar con la instalación, asegúrese de que el equipo de desarrollo cuente con las siguientes herramientas instaladas:

  • Java JDK (Versión 17 o 21): Entorno de ejecución para el servidor Spring Boot.
  • Eclipse IDE: O cualquier IDE compatible con Java (IntelliJ IDEA, VS Code).
  • Node.js y npm: Necesarios para compilar y ejecutar el proyecto Angular.
  • PostgreSQL: Motor de base de datos relacional.
  • pgAdmin 4: Interfaz gráfica para gestionar la base de datos PostgreSQL.
  • Git: Para clonar el repositorio del proyecto.

2. Configuración de la Base de Datos

LiFe System requiere una base de datos operativa antes de que el servidor Backend pueda iniciarse.

  1. Abra pgAdmin 4 e inicie sesión en su servidor local de PostgreSQL.
  2. Haga clic derecho en Databases y seleccione Create > Database.
  3. Asigne el nombre de la base de datos (por ejemplo, librarypro_db) y guárdela.
  4. Haga clic derecho sobre la base de datos recién creada y abra la herramienta Query Tool.
  5. Copie el contenido del archivo schema.sql (ubicado en la carpeta DataBase/ del proyecto) y ejecútelo para crear todas las tablas necesarias.
Figura 1: Creación de tablas en pgAdmin 4.
Importante: Anote el nombre de usuario y contraseña de su PostgreSQL, ya que los necesitará para configurar el Backend en el siguiente paso.

3. Despliegue del Backend (Java / Spring Boot)

El backend provee la API REST y se conecta directamente a la base de datos PostgreSQL.

Paso 3.1: Importar el proyecto

  1. Abra Eclipse IDE.
  2. Vaya a File > Import > Maven > Existing Maven Projects.
  3. Seleccione la carpeta AppBack/libraryPro de su repositorio y haga clic en Finish.
  4. Espere a que Maven descargue todas las dependencias listadas en el pom.xml.

Paso 3.2: Configurar credenciales

  1. En el árbol del proyecto, navegue hasta src/main/resources/application.properties.
  2. Edite el archivo para que coincida con su configuración local:
spring.datasource.url=jdbc:postgresql://localhost:5432/librarypro_db spring.datasource.username=tu_usuario_postgres spring.datasource.password=tu_contraseña_postgres

Paso 3.3: Ejecutar el Servidor

  1. Localice la clase principal AppLibraryPro.java en src/main/java/com/ssdk/libraryPro/.
  2. Haga clic derecho sobre el archivo y seleccione Run As > Java Application (o Spring Boot App).
  3. Observe la consola de Eclipse. Debería ver el mensaje de que el Tomcat embebido inició correctamente en el puerto 8080.
Figura 2: Consola de Eclipse mostrando el arranque exitoso.

4. Despliegue del Frontend (Angular)

Una vez que el backend y la base de datos están corriendo, podemos levantar la interfaz gráfica de usuario.

  1. Abra una terminal de comandos (CMD, PowerShell o Git Bash).
  2. Navegue hacia la carpeta del frontend del proyecto:
    cd ruta/hacia/pr-LibraryPro/AppFrond
  3. Instale las dependencias de Node.js (Esto solo es necesario la primera vez):
    npm install
  4. Inicie el servidor de desarrollo de Angular:
    npm run start
    Nota: Internamente, esto ejecuta ng serve --o.
Figura 3: Terminal mostrando la compilación exitosa de Angular.

5. Verificación de Funcionamiento

Para comprobar que toda la arquitectura está comunicándose correctamente:

  1. El comando de Angular debió abrir automáticamente su navegador web en http://localhost:4200.
  2. Si no lo hizo, abra su navegador y visite dicha dirección.
  3. Debe visualizar la pantalla de **Login** de LiFe System.
  4. Intente iniciar sesión con un usuario existente en su base de datos. Si ingresa al sistema sin errores de red, **la instalación ha sido un éxito.**
En caso de error: Presione F12 en su navegador, vaya a la pestaña Consola o Red (Network) y verifique si hay errores CORS o de conexión rechazada. Esto generalmente indica que el Backend de Spring Boot (paso 3) no se inició correctamente.
